LRU — Least Recently Used

Live Windows Processes
How LRU works: When a page fault occurs and all frames are full, the page that has not been used for the longest time is evicted. LRU uses an OrderedDict to track recency — on each access, the page is moved to the end. The first item is always the least recently used. Unlike FIFO, LRU does not suffer from Bélady’s Anomaly.

Live Process Connection: The reference string reflects real Windows processes. Virtual pages and time-slices are distributed using the OS's most volatile metric: CPU execution time, meaning highly active processes will dominate the page references. LRU excels here because these highly active workloads generate real temporal locality: recently accessed pages are likely to be accessed again quickly, which LRU exploits by keeping them in memory.
