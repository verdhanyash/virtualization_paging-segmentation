Optimal — Furthest Future Use

Live Windows Processes
How Optimal works: This is the theoretical best algorithm. On each page fault, it looks ahead in the reference string and evicts the page whose next use is farthest in the future (or never used again). It produces the minimum possible faults for any given reference string. It is not implementable in practice (requires future knowledge) but serves as the gold standard benchmark.

Live Process Connection: The reference string comes from real Windows processes, dynamically driven by the most volatile metric (CPU time). Compare Optimal's fault count against FIFO and LRU on this live workload — the gap reveals how much room each algorithm has for improvement. By weighting references by active CPU execution (simulating real thrashing and page fault probability), you'll see LRU typically closely trails Optimal's performance, while FIFO struggles.
