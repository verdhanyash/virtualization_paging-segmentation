Segmentation — Memory Allocation Simulator

● Variable-Size Segments
How segmentation works: Memory is divided into variable-size segments (code, stack, heap, etc.). When allocating, the OS finds a suitable free hole using a placement strategy (First-Fit, Best-Fit, Worst-Fit, or Next-Fit). Freeing a segment creates a hole. Over time, holes fragment memory — external fragmentation. Block alignment causes internal fragmentation (wasted bytes). Compaction slides segments left to merge all holes.
