FIFO — First In, First Out

Live Windows Processes
How FIFO works: Pages are loaded into frames in order. When a page fault occurs and all frames are full, the oldest page (the one that entered memory first) is evicted — like a queue. FIFO is simple but can suffer from Bélady’s Anomaly, where increasing the number of frames actually increases faults.

Live Process Connection: The reference string is generated from real Windows processes on your machine. Instead of using static memory size, virtual pages and simulated CPU time slices are assigned proportionally based on the most volatile metric: actual CPU execution time. Highly active processes like brave generating high CPU activity get more pages and reference hits than idle apps. The sequence simulates CPU scheduling with locality of reference: active processes get more time slices, and page accesses follow spatial/temporal locality patterns.
