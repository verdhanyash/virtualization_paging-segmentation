Key Concepts
Demand Paging
Pages are loaded into physical memory only when accessed — not in advance. On the first reference to a page not yet in RAM, a page fault fires, the OS fetches the page from disk, maps it via the page table, and resumes execution. This is exactly what every simulation on this site models: a reference string drives page accesses, and the replacement algorithm decides which frame to evict when memory is full.
Page Fault
When a referenced page is not in any frame, a page fault occurs. The OS must load it from disk — an expensive operation.
Bélady's Anomaly
A counter-intuitive phenomenon in FIFO where increasing the number of frames can increase page faults for certain reference strings.
External Fragmentation
Free memory is split into small, non-contiguous holes between allocated segments — enough total free space, but no single hole is large enough.
